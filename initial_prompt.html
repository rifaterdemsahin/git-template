Usage Guide Copilot Initial Prompt Flow

Copilot Initial Prompt

After creating your new repository from this template, the first thing you should do is send GitHub Copilot a structured initial prompt that gives it full context about your project. A well-crafted first message dramatically improves every subsequent Copilot suggestion.

Why send an initial prompt?

GitHub Copilot has no knowledge of your project goals unless you tell it. By sending a single, clear prompt right after cloning, you:

  • Give Copilot the project name, purpose, and tech stack.
  • Set the coding conventions you want it to follow.
  • Establish the tone and scope so suggestions stay on target.

The initial prompt command

Open GitHub Copilot Chat (VS Code sidebar or Ctrl+Shift+I) and paste the following prompt, replacing the placeholder values with your own details: 

You are helping me build a project called [PROJECT_NAME].

Tech stack: [e.g. TypeScript, React, Node.js, PostgreSQL]
Purpose: [one-sentence description of what the project does]
Coding conventions:
  - Use [tabs / 2-space / 4-space] indentation
  - Prefer [async/await / Promises / callbacks]
  - Follow [Airbnb / Google / Standard] style guide
  - Write tests with [Jest / Vitest / Mocha]

Please keep all suggestions consistent with these conventions
and ask me before introducing any new dependency.
Tip: You can store this prompt in a file called .github/copilot-instructions.md in your repository. Copilot Chat automatically picks it up as workspace instructions, so you never have to paste it manually again.

Example filled-in prompt

You are helping me build a project called my-new-project.

Tech stack: TypeScript, Node.js, Express, SQLite
Purpose: A REST API that tracks personal reading goals.
Coding conventions:
  - Use 2-space indentation
  - Prefer async/await
  - Follow the Airbnb style guide
  - Write tests with Jest

Please keep all suggestions consistent with these conventions
and ask me before introducing any new dependency.

Using .github/copilot-instructions.md

To make the instructions permanent and shareable with your whole team, create the file in your repository: 

mkdir -p .github
cat > .github/copilot-instructions.md << 'EOF'
You are helping me build a project called my-new-project.

Tech stack: TypeScript, Node.js, Express, SQLite
Purpose: A REST API that tracks personal reading goals.
Coding conventions:
  - Use 2-space indentation
  - Prefer async/await
  - Follow the Airbnb style guide
  - Write tests with Jest

Please keep all suggestions consistent with these conventions
and ask me before introducing any new dependency.
EOF
Note: The copilot-instructions.md feature requires GitHub Copilot Chat and a compatible editor extension (VS Code v1.90+ or JetBrains 2024.1+). See the GitHub Docs for the full reference.
