git-template

A reusable CI skeleton you can stamp onto every new project with a single gh repo create --template command — no more copy-pasting boilerplate.

What is gh repo create --template?

The GitHub CLI (gh) lets you create a new repository that is pre-populated with all the files from a template repository. The --template flag accepts any OWNER/REPO slug that has been marked as a Template repository in GitHub settings.

Prerequisite: install the GitHub CLI and authenticate once with gh auth login.

Quick-start example

Run the command below to spin up a new repository called my-new-project using this template: 

# Create a new public repository from the template
gh repo create my-new-project \
  --template rifaterdemsahin/git-template \
  --public \
  --clone

# Enter the freshly cloned directory and start building
cd my-new-project

What each flag does:

  • my-new-project — name of the repository to create.
  • --template rifaterdemsahin/git-template — copy all files from this template.
  • --public — make the new repository public (use --private for a private repo).
  • --clone — automatically clone the new repo to your local machine.

What you get

After running the command you will have a ready-to-use project with:

  • A CI skeleton pre-wired and ready to extend.
  • No history from the template — your project starts with a clean slate.
  • All template files committed as the initial commit.

How to mark this repo as a template

To enable the --template flag for a repository, go to Settings → General and tick the Template repository checkbox. This makes the repo visible to gh repo create --template for any user with read access.
